# Installer reference

The `webtrees-installer` wizard renders a self-contained `compose.yaml`
+ `.env` (and brings the stack up when asked). This page is a
scenario-oriented reference; for the bare flag list run
`./install --help`.

The flow always renders into `--work-dir` (default: the current
directory). Re-running the wizard with `--force` overwrites the
existing files; without `--force` the wizard refuses to clobber.

## First-time install

The path most operators land on via the one-liner. Interactive
prompts cover everything the wizard cannot infer:

```bash
./install
```

Equivalent fully-flagged form for documentation / scripts:

```bash
./install \
    --work-dir /srv/webtrees \
    --edition full \
    --proxy standalone \
    --port 28080
```

The wizard renders `compose.yaml` + `.env`, generates an admin
password (printed once in the install banner), then runs
`docker compose up -d`. Browser-land at `http://<host>:28080`.

| Flag | Effect |
|---|---|
| `--work-dir <path>` | Target directory for the rendered files. Default `/work` (the installer container's bind-mount). |
| `--edition core\|full` | Picks the `webtrees` vs `webtrees-full` image. `full` ships the bundled module catalogue. |
| `--proxy standalone\|traefik` | Reverse-proxy mode. `standalone` publishes a host port; `traefik` emits Traefik labels and joins the `traefik` external network. |
| `--port <N>` | Host port nginx binds in standalone mode. Default 28080; falls back to 28081 on first conflict, then prompts. |
| `--domain <host>` | Public domain for Traefik mode. Required when `--proxy traefik`. |

## Re-install with new options

The wizard tolerates re-runs on an existing workdir as long as
`--force` is passed. The DB volume and admin credentials survive
because they live in the docker namespace, not on the rendered files:

```bash
./install --force --edition full
```

Combine with `--no-up` to render-and-stop so the operator can review
the diff before bringing the stack up:

```bash
./install --force --edition full --no-up
docker compose up -d
```

If the previous install left volumes that should not survive (e.g. a
test run that landed an admin you cannot log in as), the wizard
detects them and offers an interactive wipe — see
[customizing.md → Surviving-volume detection](customizing.md). The
non-interactive path requires the operator to wipe manually:
`docker compose -p <project> down -v` before re-running.

## Headless / non-interactive

Every prompt has a CLI override; passing `--non-interactive` makes
the wizard fail loud on any missing answer rather than asking:

```bash
./install \
    --non-interactive \
    --force \
    --edition full \
    --proxy standalone \
    --port 28080 \
    --admin-user admin \
    --admin-email admin@example.com
```

| Flag | Effect |
|---|---|
| `--non-interactive` | Refuse to prompt. Every required answer must be supplied via flag. |
| `--admin-user <name>` | Headless admin bootstrap. Default `admin` with an autogenerated random password the install banner prints once. |
| `--admin-email <addr>` | Admin email. Default `admin@example.org`. |
| `--no-admin` | Skip the admin bootstrap entirely; the stack starts in webtrees' upstream setup wizard, which asks for DB credentials interactively over HTTP. Use only when you intend to drive the upstream wizard yourself. |
| `--no-up` | Render `compose.yaml` + `.env` and stop. `docker compose up -d` is left to the operator. |

The autogenerated admin password is shown in the banner exactly once
and not written to disk. If the operator misses it, the recovery path
is to wipe the secrets volume and re-run the install.

## Behind a reverse proxy (Traefik)

The bundled Traefik overlay terminates TLS at the edge; the rendered
router pins `websecure` + `tls=true`. The wizard rejects
`--no-https --proxy traefik` because the combination produces an
internally inconsistent stack — see
[customizing.md → HTTPS trust gate](customizing.md).

```bash
./install \
    --proxy traefik \
    --domain webtrees.example.com \
    --edition full \
    --admin-user admin \
    --admin-email admin@example.com
```

The render assumes a Traefik instance running on the `traefik`
external Docker network. The trust-proxy gate in
`rootfs/etc/nginx/includes/trust-proxy-map.conf` accepts
`X-Forwarded-Proto: https` only from `172.16.0.0/12` (Docker's default
user-bridge range). Operators on custom Docker networks outside that
range need the bind-mount workaround documented in customizing.md.

## Dev mode

The `--mode dev` path is for working on `webtrees-docker` itself or on
modules against this stack. It bind-mounts `./app` into phpfpm and
writes `.env` against the in-repo `compose.yaml` / overlay chain
rather than a freshly-rendered standalone one:

```bash
./install --mode dev
./install --mode dev --dev-domain webtrees.local
./install --mode dev --use-external-db --external-db-host db.example.com
```

Dev-only flags:

| Flag | Effect |
|---|---|
| `--mode dev` | Switch from standalone-install mode to in-repo dev mode. |
| `--dev-domain <host>` | Hostname the dev stack binds (Traefik-routed). Defaults to `localhost:<port>` in standalone proxy mode. |
| `--pma-port <N>` | Host port phpMyAdmin binds. Default 50011. |
| `--mariadb-root-password <pw>` | Root DB password for the bundled MariaDB. Random when omitted. |
| `--mariadb-database <name>` / `--mariadb-user <name>` / `--mariadb-password <pw>` | App-side DB credentials. |
| `--use-existing-db` | Assume the DB is already initialised — skip the schema-init step. |
| `--use-external-db` | Skip the bundled MariaDB service and pull `compose.external.yaml` into the chain. |
| `--external-db-host <host>` | Hostname the app uses with `--use-external-db`. |
| `--local-user-id <uid>` / `--local-user-name <name>` | Host UID/username for the bind-mount UID-remap. The `./install` launcher fills these from the calling shell; pass explicitly only when bypassing the launcher. |

## Cross-cutting flags

| Flag | Effect |
|---|---|
| `--version` | Print the installer version and exit. |
| `--force` | Overwrite an existing `compose.yaml` / `.env` without prompting. Required for re-installs. |
| `--no-https` | Standalone proxy mode only. Set `ENFORCE_HTTPS=FALSE` in the rendered stack. Rejected with `--proxy traefik`. |
| `--pretty-urls` | Standalone proxy mode only. Render `WEBTREES_REWRITE_URLS=1` so tree pages serve as `/tree/.../individual/...` rather than `?route=...`. Dev mode honours the value in `.env` directly. |
| `--demo` | Generate a 7-generation synthetic GEDCOM family tree and (if the stack is up) import it. |
| `--demo-seed <N>` | RNG seed for the demo tree. Default 42 — same seed produces the same tree. |

## Exit codes

| Code | Meaning |
|---|---|
| 0 | Success |
| 1 | User cancelled an interactive prompt |
| 2 | `PrereqError` or `PromptError` — missing input, missing host tool, daemon down |
| 3 | `StackError` — `docker compose up` failed mid-flow |
| 4 | Pull / install failure in a downstream step |

See `installer/webtrees_installer/cli.py:_run_with_exit_codes` for the
authoritative translation table.
